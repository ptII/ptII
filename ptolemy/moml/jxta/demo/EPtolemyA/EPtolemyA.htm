EPtolemyA

This demo uses JXTA to share actors between two instances of Ptolemy II. To run the demo, run 
make demo
in one window and 
cd $PTII/ptolemy/moml/jxta/demo/EPtolemyB
make demo
in another window. The first time you run the demos set the HTTP and TCP address(in the "Advanced" tab of the JXTA Configuration window) to be different for these two peers to make them work on one machine.

Notes

  • To reset the JXTA Configuration, run 
    make jxtaclean
    which will remove the .jxta/ directories both demo directories.
  • It might take some time at the first time for the configuration. For the first time, the configuration will try to contact a server(hard-coded by the current JXTA implementation) to get some default rendezvous address. (This is not a good design, but let's just bear it for now.) Sometimes, the server may not work, and you'll get an message as: 
      Error: cannot instantiate peer group.......
    If so, try it later.
  • Note that this demo requires an Internet connection because it tries to connect to a JXTA server.
  • The second time the demo is run, a small window will pop up asking for a name and password. Unfortunately, sometimes this window is obscured by other windows, so you may have to cycle through all the windows to find the name/password window.
